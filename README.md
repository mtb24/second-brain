# Second Brain

A personal operating system running on a VPS. Captures thoughts, runs an AI agent (Cortex), and hosts a live trading strategy tournament where Claude-powered bots compete against each other using real market data.

---

## What it is

Second Brain is a self-hosted system with four main capabilities:

- **Thought ingestion** — capture notes, ideas, and context via Telegram or API. Stored with vector embeddings for semantic search.
- **AI agent (Cortex)** — OpenClaw agent on the VPS (Telegram + Mission Control). Model routing is configured in OpenClaw (e.g. Google / Groq / local Ollama); see **BRAIN.md** → OpenClaw. Tournament bots may still use Anthropic separately.
- **Trading tournament** — multiple Claude-powered bots compete in timed rounds using real CoinGecko market data. Each bot follows a different strategy (momentum, mean-reversion, breakout, etc.) and makes independent buy/sell/hold decisions. Results are tracked on a live leaderboard.
- **Mission Control** — a web dashboard at `mission.kendowney.com` that shows system health, thought history, semantic search, agent status, and the tournament.

## Tournament system

The tournament runs Claude-powered trading bots in parallel. Each bot receives identical real market data (BTC, ETH from CoinGecko) and makes independent decisions using a different strategy document.

### How a round works

1. Orchestrator selects active strategies from the DB (default: 3 bots)
2. If `BOT_COUNT` exceeds available strategies, Claude researches and writes new ones
3. All bots start simultaneously with equal balance ($10,000)
4. Every 60 seconds, each bot receives current market data and calls Claude to decide: buy / sell / hold / close
5. Decisions and tick snapshots are logged to the DB in real time
6. After the round (configurable: 10m / 30m / 1h / 3h), final performance is recorded
7. Winner is promoted; underperformers below the fitness threshold are retired

### Strategies

Three seed strategies are included:

- **momentum** — buys when price is trending up, sells when trending down
- **mean-revert** — buys below moving average, sells above
- **breakout** — buys on volume breakout above resistance

New strategies are auto-generated by Claude when needed.

### MockAdapter

The mock exchange adapter simulates real trading:
- Prices seeded from real CoinGecko data on first access
- Random walk applied (±0.1% per tick) to simulate price movement
- Positions tracked with full P&L calculation
- `DRY_RUN` guard only applies to live adapters — mocks always execute

## Cortex (AI Agent)

Cortex runs inside **OpenClaw** on the VPS. Primary/fallback models are configured in **`~/.openclaw/openclaw.json`** (see **BRAIN.md** → OpenClaw). Accessible via:
- Telegram bot `@kensbrainbot`
- Mission Control → OpenClaw page
- Direct WebSocket connection

Cortex can read and write files, run shell commands, query the DB, and push to GitHub. It follows strict rules in `AGENTS.md` — it refuses unsupervised autonomous action and will not push credentials.

## Local development

**Prerequisites:** Node 22, Python 3.12, Docker, SSH access to the VPS.

```bash
git clone https://github.com/mtb24/second-brain.git
cd second-brain
cp .env.example .env   # fill in credentials
```

**Run the tournament locally:**
```bash
cd tournament
npm install
npm run tournament
```

**Deploy changes to the server:**
```bash
# Rsync mission-control source
rsync -av --delete mission-control/ brain@147.182.240.24:~/brain/mission-control/

# Rebuild and restart
ssh brain@147.182.240.24 "cd ~/brain && docker compose build --no-cache mission-control && docker compose up -d mission-control"
```

**Sync server changes to local:**
```bash
git pull   # Cortex commits and pushes changes directly from the server
```

## Environment variables

All credentials live in a single `.env` file at `~/brain/.env` on the server and `/path/to/second-brain/.env` locally. See `.env.example` for the full list.

Key variables:

| Variable | Purpose |
|----------|---------|
| `POSTGRES_*` | Database connection |
| `API_SECRET` | Ingest API authentication |
| `MCP_SECRET` | MCP server authentication |
| `ANTHROPIC_API_KEY` | Claude API (used by Cortex and tournament bots) |
| `COINGECKO_API_KEY` | Market data for tournament |
| `OPENCLAW_GATEWAY_URL` | WebSocket endpoint for Cortex |
| `TOURNAMENT_BOT_COUNT` | Number of bots per round (default: 3) |
| `DRY_RUN` | Set to `false` to enable live trading (default: `true`) |

## Known gotchas

- **OpenClaw auth** — uses trusted-proxy mode. Mission Control must connect via `wss://mission.kendowney.com/openclaw/` through Nginx, not directly. Direct connections will be rejected.
- **Docker bridge IP** — `brain_default` network gateway is `172.19.0.1`
- **Vector extension** — `initdb/01-init.sql` auto-installs on fresh DB. If DB already exists: `docker exec brain-db psql -U brain -d brain -c "CREATE EXTENSION IF NOT EXISTS vector;"`
- **Tournament tables** — not in `initdb/01-init.sql`. Must be created via migration after a fresh DB wipe.
- **`B2_APP_KEY`** — never `B2_APPLICATION_KEY`
